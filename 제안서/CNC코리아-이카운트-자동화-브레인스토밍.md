## 업무 흐름 패턴 분석

### 공통 패턴 식별

```
[외부 데이터 소스] → [파싱/변환] → [이카운트 입력] → [문서 생성] → [메일 발송]
```

#### 데이터 입력 경로 (5가지)
1. **카카오톡 메시지** → 주문 정보 추출
2. **PG사 엑셀** (이니시스, 네이버페이) → 매출 데이터
3. **카페24 관리자** → 발주 내역
4. **PDF 문서** (인보이스, 정산서, 관세) → 구매 데이터
5. **엑셀 파일** (급여대장, 채권현황) → 내부 데이터

#### 데이터 출력 경로 (4가지)
1. **이카운트 ERP 입력** (판매, 구매, 매출전표, 입출금)
2. **엑셀 파일 생성** (출하지시, 발주서, 급여이체현황서, 채권현황)
3. **HWP 파일 생성** (급여이체신청서)
4. **메일 발송** (물류센터, 거래처, IBK, 개인별)

## 3단계 기획안

### 기획 1: Chrome 확장프로그램 (이카운트 직접 연동)

#### 핵심 컨셉
> 이카운트 웹 페이지 위에서 동작하는 Chrome Extension으로, 사용자의 브라우저 세션을 활용해 이카운트 DOM을 직접 조작하여 데이터 입력 자동화

#### 기능 범위
- **자동 로그인 및 세션 유지**
- **판매입력 자동화**: 카톡 메시지 복붙 → 파싱 → 이카운트 판매입력 폼 자동 채움
- **매출전표 자동 입력**: PG사 엑셀 업로드 → 자동 분류 → 전표 일괄 입력
- **구매입력 자동화**: PDF 드래그&드롭 → OCR/파싱 → 구매전표 자동 생성
- **입출금 대사**: 입금 내역 업로드 → 매출 내역과 자동 매칭 → 미수금 표시
- **출하지시서 자동 생성**: 판매 데이터 기반 엑셀 자동 다운로드

#### 기술 스택
- Chrome Extension Manifest V3
- Content Script (이카운트 DOM 조작)
- Background Service Worker
- PDF.js / Tesseract.js (PDF 파싱/OCR)
- SheetJS (엑셀 읽기/쓰기)

#### 장점
- 빠른 개발 (2~3개월)
- 이카운트 API 불필요 (DOM 기반)
- 사용자 기존 환경 그대로 사용
- 낮은 도입 비용

#### 리스크
- 이카운트 UI 변경 시 유지보수 필요
- DOM 의존으로 안정성 이슈
- 브라우저 한정 (데스크톱만)

#### 예상 비용: ★☆☆ (가장 저렴)

---

### 기획 2: MCP 연동 플랫폼 (데이터 허브 + 보고서)

#### 핵심 컨셉
> 외부 데이터 소스(PG사, 카페24, 카톡, PDF)를 수집하여 이카운트에 자동 반영하고, 축적된 데이터로 보고서/통계를 자동 생성하여 관리자에게 정기 발송하는 **중앙 데이터 허브**

#### 기능 범위

##### 2-1. 데이터 수집 모듈
- **PG사 연동**: 이니시스 API / 네이버페이 API 연동
- **카페24 연동**: 카페24 관리자 API 연동
- **카카오톡 수집**: 카톡 봇 또는 웹훅 기반 주문 수집
- **PDF 자동 파싱**: 인보이스, 정산서, 관세 문서 OCR + AI 구조화
- **엑셀 업로드**: 급여대장, 채권현황 등 수동 업로드 지원

##### 2-2. 이카운트 자동 반영
- **이카운트 API 연동** (또는 RPA 방식)
- 판매입력, 구매입력, 매출전표, 입출금 자동 반영
- 데이터 검증 및 오류 알림

##### 2-3. 문서 자동 생성 & 발송
- **견적서 생성**: 템플릿 기반 PDF/HTML 견적서
- **발주서 생성**: 카페24 데이터 → 엑셀 발주서 자동 작성
- **출하지시서**: 판매 데이터 → 물류센터용 엑셀
- **급여 문서**: 급여이체현황서(엑셀) + 급여이체신청서(HWP) + 개인별 명세서
- **메일 자동 발송**: SMTP 연동 → 물류/거래처/IBK/개인 메일

##### 2-4. 보고서 & 통계 대시보드 ⭐ 핵심 차별화
- **일별 보고서**: 당일 매출, 입금, 미수금 현황
- **주간 보고서**: 주간 매출 추이, 채권 현황, 발주 현황
- **월별 보고서**: 월간 매출/매입 종합, 손익 요약, 재고 현황
- **실시간 대시보드**: 매출 추이 차트, 거래처별 분석, 제품별 분석
- **자동 리포트 발송**: 담당자/대표자에게 일별/주간/월별 메일 자동 발송

#### 기술 스택
- **백엔드**: Node.js/Python + FastAPI
- **프론트엔드**: Next.js 대시보드
- **데이터**: PostgreSQL + Redis
- **AI**: Claude API (PDF 파싱, 데이터 구조화, 보고서 생성)
- **MCP 서버**: 이카운트/PG사/카페24 각각 MCP 서버 구축
- **메일**: Nodemailer / AWS SES
- **배포**: AWS / Vercel

#### 장점
- 확장성 높음 (다른 고객사에도 적용 가능)
- 보고서/통계 기능으로 경영 의사결정 지원
- API 기반으로 안정적
- 모바일 대시보드 제공 가능

#### 리스크
- 이카운트 API 제약 (공식 API 범위 확인 필요)
- 개발 기간 상대적으로 김 (4~6개월)
- 인프라 운영 비용

#### 예상 비용: ★★☆ (중간)

### 기획 3: 독립 ERP 플랫폼 (이카운트 대체)

#### 핵심 컨셉
> CNC코리아의 업무 프로세스에 최적화된 **맞춤형 ERP 플랫폼**을 구축하여 이카운트를 완전 대체. AI 기반 자동화가 내장된 차세대 ERP

#### 기능 범위

##### 3-1. 핵심 ERP 모듈
- **판매관리**: 주문 접수 → 판매입력 → 출하지시 → 배송 추적
- **구매관리**: 발주 → 입고 → 구매전표 → 대금 지급
- **재고관리**: 입출고 관리, 재고 실사, 안전재고 알림
- **회계관리**: 매출전표, 매입전표, 입출금, 채권/채무
- **거래처관리**: 거래처 정보, 거래 이력, 신용 관리

##### 3-2. AI 내장 자동화
- 카카오톡 주문 → 자동 판매입력 (NLP 기반)
- PG사 데이터 → 자동 매출전표 (규칙 + AI)
- PDF 문서 → 자동 구매입력 (AI OCR)
- 입금 → 매출 자동 대사 (AI 매칭)
- 채권 현황 자동 업데이트
- AI 기반 수요 예측, 재고 최적화

##### 3-3. 커뮤니케이션 통합
- 카카오톡 비즈 연동 (주문 접수, 알림)
- 이메일 자동 발송 (견적서, 발주서, 급여 등)
- Slack/Teams 알림

##### 3-4. 보고서 & BI
- 실시간 대시보드 (매출, 재고, 채권)
- AI 기반 인사이트 (이상 탐지, 트렌드 분석)
- 맞춤형 보고서 빌더
- 자동 정기 보고서 발송

#### 기술 스택
- **프론트엔드**: Next.js + Tailwind + shadcn/ui
- **백엔드**: Node.js/Python + 마이크로서비스
- **데이터**: PostgreSQL + MongoDB + Redis
- **AI**: Claude API + 자체 ML 모델
- **인프라**: AWS (ECS + RDS + S3 + SES)
- **모바일**: React Native / PWA

#### 장점
- 완전한 맞춤화 가능
- AI 네이티브 ERP (시장 차별화)
- 이카운트 의존 탈피
- SaaS 사업화 가능 (다른 중소기업에 판매)

#### 리스크
- 개발 기간 장기 (8~12개월)
- 높은 초기 투자 비용
- 기존 이카운트 데이터 마이그레이션
- 유지보수/운영 인력 필요

#### 예상 비용: ★★★ (가장 높음)
