## 1. 제안 배경

### 1.1 현황 진단

CNC코리아는 현재 이카운트 ERP를 중심으로 판매, 구매, 회계 업무를 수행하고 있으며, 다수의 외부 시스템(카카오톡, 이니시스, 네이버페이, 카페24)에서 데이터를 수동으로 수집하여 이카운트에 입력하는 반복 업무가 발생하고 있습니다.

### 1.2 주요 Pain Point

| 문제 | 영향 | 발생 빈도 |
|------|------|-----------|
| 카톡 주문 → 이카운트 수작업 입력 | 입력 오류, 시간 소요 | 매일 |
| PG사 매출 데이터 수동 전표 입력 | 분류 오류, 누락 가능 | 매일 |
| PDF 인보이스 수동 데이터 입력 | OCR 불가, 타이핑 시간 | 수시 |
| 입금-매출 수동 대사 | 미수금 파악 지연 | 매일 |
| 급여/견적 문서 수작업 생성 | 서식 불일치, 반복 작업 | 월/수시 |
| 경영 현황 보고서 부재 | 의사결정 데이터 부족 | 상시 |

### 1.3 기대 효과

- **업무 시간 절감**: 반복 입력 업무 80% 이상 자동화
- **데이터 정확도 향상**: 수작업 입력 오류 제거
- **실시간 경영 현황**: 매출/재고/채권 실시간 파악
- **의사결정 고도화**: AI 기반 트렌드 분석 및 예측

## 3. 기획 1 — Chrome 확장프로그램

### 3.1 개요

이카운트 웹 페이지 위에서 동작하는 Chrome Extension으로, 기존 업무 환경을 변경하지 않으면서 데이터 입력을 자동화합니다.

### 3.2 핵심 기능

#### F1. 매출전표 자동 입력
```
[이니시스/네이버페이 엑셀] → [Chrome 확장 업로드]
     → [카드매출/현금영수증/기타매출 자동 분류]
     → [이카운트 매출전표 자동 입력]
```
- PG사 엑셀 파일을 확장프로그램에 드래그&드롭
- AI가 결제수단별 자동 분류 (카드매출, 현금영수증, 기타매출)
- 이카운트 매출전표 화면에 자동 입력 (사용자 최종 확인 후 저장)

#### F2. 판매입력 + 출하지시
```
[카카오톡 주문 텍스트] → [복사-붙여넣기]
     → [주문 정보 파싱 (제품명, 수량, 거래처)]
     → [이카운트 판매입력 자동 채움]
     → [출하지시서 엑셀 자동 생성 + 메일 발송]
```

#### F3. 구매입력 (PDF → 이카운트)
```
[Proforma Invoice PDF / 포워딩 정산서 / 관세 PDF]
     → [AI OCR + 구조화]
     → [이카운트 구매입력 자동 채움]
```

#### F4. 입출금 대사
```
[입금내역 엑셀/CSV] → [매출내역 자동 매칭]
     → [매칭 결과 표시 (일치/불일치/미수금)]
     → [이카운트 입금 반영]
```

#### F5. 문서 생성 + 메일 발송
- 견적서 생성 (HTML/PDF)
- 발주서 생성 (엑셀)
- 급여이체현황서 (엑셀) + 급여이체신청서 (HWP)
- 개인별 급여명세서 → 개인 메일 발송

### 3.3 기술 구현

| 구성요소 | 기술 | 역할 |
|----------|------|------|
| Content Script | JavaScript | 이카운트 DOM 조작/데이터 입력 |
| Popup UI | React + Tailwind | 파일 업로드/설정 화면 |
| Background Worker | Service Worker | AI API 호출, 파일 처리 |
| PDF 파싱 | PDF.js + Claude API | PDF → 구조화 데이터 |
| 엑셀 처리 | SheetJS | 엑셀 읽기/쓰기 |
| 메일 발송 | Gmail API / SMTP | 자동 메일 발송 |

### 3.4 개발 일정

| 주차 | 마일스톤 |
|------|----------|
| 1~2주 | 이카운트 DOM 분석, Chrome Extension 뼈대 구축 |
| 3~4주 | F1 매출전표 자동 입력 개발 |
| 5~6주 | F2 판매입력 + 출하지시 개발 |
| 7~8주 | F3 PDF → 구매입력 개발 |
| 9~10주 | F4 입출금 대사 + F5 문서 생성 |
| 11~12주 | 통합 테스트, 사용자 교육, 배포 |

## 4. 기획 2 — MCP 연동 플랫폼

### 4.1 개요

외부 데이터를 자동 수집하여 이카운트에 반영하고, 축적된 데이터로 **경영 보고서와 통계 대시보드**를 제공하는 중앙 데이터 허브입니다.

### 4.2 시스템 아키텍처

```
┌──────────────────────────────────────────────────┐
│                 MCP 연동 플랫폼                     │
│                                                    │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐  │
│  │ 데이터 수집  │  │ 데이터 처리  │  │ 데이터 출력  │  │
│  │            │  │            │  │            │  │
│  │ - 이니시스   │→ │ - AI 파싱   │→ │ - 이카운트   │  │
│  │ - 네이버페이 │  │ - 분류/매칭  │  │ - 엑셀/PDF  │  │
│  │ - 카페24    │  │ - 검증      │  │ - 메일 발송  │  │
│  │ - 카카오톡   │  │ - 변환      │  │ - 대시보드   │  │
│  │ - PDF 문서  │  │            │  │ - 보고서    │  │
│  │ - 은행 데이터│  │            │  │            │  │
│  └────────────┘  └────────────┘  └────────────┘  │
│                                                    │
│  ┌─────────────────────────────────────────────┐  │
│  │          보고서 & 통계 엔진                     │  │
│  │  일별/주간/월별 자동 리포트 → 메일 발송          │  │
│  └─────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────┘
```

### 4.3 핵심 기능

#### 모듈 A: 데이터 자동 수집
- **PG사 연동**: 이니시스/네이버페이 API 또는 파일 자동 수집
- **카페24 연동**: 주문/발주 데이터 API 연동
- **카카오톡 봇**: 단체방 주문 메시지 자동 수집 및 파싱
- **PDF 수집함**: 이메일 첨부 또는 업로드 → AI 자동 파싱
- **은행 데이터**: 기업은행 입금내역 자동 수집 (파일 기반)

#### 모듈 B: 이카운트 자동 반영
- MCP 서버를 통한 이카운트 데이터 입력 자동화
- 판매입력, 구매입력, 매출전표, 입출금 일괄 처리
- 데이터 검증 → 오류 건 알림 → 사용자 승인 후 확정

#### 모듈 C: 문서 자동 생성 & 발송
| 문서 | 형식 | 수신처 |
|------|------|--------|
| 출하지시서 | 엑셀 | 물류센터 |
| 발주서 | 엑셀 | 거래처 |
| 견적서 | PDF/HTML | 거래처 |
| 급여이체현황서 | 엑셀 | 기업은행 |
| 급여이체신청서 | HWP | 기업은행 |
| 급여명세서 | PDF | 개인별 |
| 채권현황 리스트 | 엑셀 | 내부 |

#### 모듈 D: 보고서 & 통계 대시보드 ⭐

##### 자동 보고서 발송
| 보고서 | 주기 | 수신자 | 주요 내용 |
|--------|------|--------|-----------|
| 일간 현황 | 매일 09:00 | 담당자, 대표 | 전일 매출, 입금, 미수금, 출하 현황 |
| 주간 리포트 | 매주 월요일 | 담당자, 대표 | 주간 매출 추이, 채권 현황, 발주/입고 현황 |
| 월간 보고서 | 매월 1일 | 대표 | 월간 매출/매입 종합, 손익 요약, 재고 현황, 거래처별 분석 |
| 채권 현황 | 매주 목요일 | 담당자 | 채권 리스트, 입금 확인, 미수금 aging |

##### 실시간 대시보드
- 매출 추이 차트 (일별/주별/월별)
- 거래처별 매출 분석
- 제품별 판매 현황
- 미수금/채권 현황
- 재고 현황 및 알림
- 수입 원가 분석

### 4.4 기술 스택

| 레이어 | 기술 | 비고 |
|--------|------|------|
| 프론트엔드 | Next.js + Tailwind + Recharts | 대시보드/보고서 뷰어 |

## 5. 기획 3 — 독립 ERP 플랫폼

### 5.1 개요

CNC코리아의 업무 프로세스를 완벽 반영한 **AI 네이티브 맞춤형 ERP**를 구축합니다. 이카운트를 완전히 대체하며, 향후 SaaS 제품으로 사업화가 가능합니다.

### 5.2 시스템 아키텍처

```
┌────────────────────────────────────────────────────┐
│              AI 네이티브 ERP 플랫폼                   │
│                                                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │ 판매관리  │ │ 구매관리  │ │ 재고관리  │ │ 회계   │ │
│  │ - 주문    │ │ - 발주   │ │ - 입출고  │ │ - 전표 │ │
│  │ - 출하    │ │ - 입고   │ │ - 실사   │ │ - 원장 │ │
│  │ - 반품    │ │ - 검수   │ │ - 이동   │ │ - 결산 │ │
│  └──────────┘ └──────────┘ └──────────┘ └────────┘ │
│                                                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐ │
│  │ 거래처    │ │ 급여/인사 │ │ 문서관리  │ │ BI/AI  │ │
│  │ - CRM    │ │ - 급여   │ │ - 견적서  │ │ - 대시  │ │
│  │ - 이력   │ │ - 명세서  │ │ - 발주서  │ │  보드  │ │
│  │ - 채권   │ │ - 이체   │ │ - 계약서  │ │ - 예측 │ │
│  └──────────┘ └──────────┘ └──────────┘ └────────┘ │
│                                                      │
│  ┌──────────────────────────────────────────────┐   │
│  │              AI 자동화 엔진                     │   │
│  │  카톡 NLP │ OCR │ 자동 분류 │ 대사 │ 예측      │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  ┌──────────────────────────────────────────────┐   │
│  │              외부 연동                          │   │
│  │  이니시스 │ 네이버페이 │ 카페24 │ IBK │ 카카오   │   │
│  └──────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────┘
```

### 5.3 차별화 포인트 (vs 이카운트)

| 항목 | 이카운트 | 제안 플랫폼 |
|------|----------|-------------|
| 데이터 입력 | 100% 수작업 | AI 자동 입력 (수작업 20% 이하) |
| 외부 연동 | 제한적 | PG사/카페24/카톡/은행 통합 |
| 보고서 | 기본 출력 | AI 분석 보고서 + 자동 발송 |
| 문서 생성 | 별도 작업 | 시스템 내 자동 생성/발송 |
| 입금 대사 | 수작업 | AI 자동 매칭 |
| 채권 관리 | 수동 확인 | 자동 추적 + 알림 |
| 모바일 | 제한적 | 반응형 웹 + PWA |
| 카카오톡 | 미연동 | 주문 접수/알림 통합 |

### 5.4 SaaS 사업화 가능성

CNC코리아를 첫 번째 고객으로 검증한 후, 유사 업종(수입/유통/도소매)의 중소기업에 SaaS로 제공 가능:

- **타겟 시장**: 이카운트 사용 중소기업 (국내 약 10만 개사)
- **가격 모델**: 월 구독료 (사용자 수 / 모듈 기준)
- **차별화**: AI 자동화 내장 ERP (시장에 없는 제품)

### 5.5 개발 일정

| 월 | 마일스톤 |
|----|----------|
| 1~2개월 | 요구사항 분석, 아키텍처 설계, DB 스키마 |
| 3~4개월 | 핵심 ERP 모듈 (판매, 구매, 회계) |
| 5~6개월 | 재고, 거래처, 문서 관리 모듈 |
| 7~8개월 | AI 엔진, 외부 연동, 대시보드 |
| 9~10개월 | 급여/인사, BI/보고서 모듈 |
