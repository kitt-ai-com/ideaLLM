> [!takeaway] 핵심 테이크어웨이
> 마케팅/광고 대행사 입장에서 Web Clipper는 단순 북마크 대체가 아니라 **업계 동향·경쟁사 자료·클라이언트 인사이트를 LLM 위키(`raw/`)에 자동 공급하는 수집 레이어**다. 지금 운영 중인 ideaLLM 위키의 3층 구조(Schema-Wiki-Raw)와 정확히 맞물려, 웹 → `Clippings/` → 정리 요청 한 줄 → 카테고리별 위키 페이지로 흐르는 파이프라인을 만들 수 있다. 이게 누적되면 AI에게 답을 시킬 때마다 우리 회사 컨텍스트가 깊어진다 — "인터넷 소비"가 아니라 "인터넷 자산화".

# Obsidian Web Clipper 활용법

## 개요

Obsidian Web Clipper는 웹페이지를 마크다운으로 변환해 Obsidian Vault에 저장하는 공식 브라우저 확장이다. 북마크는 링크만 남기지만 Web Clipper는 본문·메타데이터(제목·작성자·URL·요약·태그)를 함께 보관해 **검색 가능한 장기 지식 자산**으로 만든다. 사이트가 사라져도 자료는 남는다.

## Web Clipper vs 일반 북마크

| 구분 | 일반 북마크 | Obsidian Web Clipper |
|---|---|---|
| 저장 방식 | URL만 | 본문 마크다운 + 메타데이터 |
| 검색 | 불가 | 전문(full-text) 검색 |
| 재활용 | 낮음 | 노트 재구성·인용·교차참조 |
| 장기성 | 원본 사이트 사라지면 끝 | Vault에 영구 보존 |
| 구조화 | 없음 | 템플릿·태그 자동 |
| AI 연계 | 없음 | LLM 컨텍스트·장기기억으로 활용 |

## 설치 및 사용 흐름

1. Chrome/Edge에 Obsidian Web Clipper 확장 설치
2. Obsidian Vault 실행 (이 프로젝트의 경우 `ideaLLM/`)
3. 저장할 웹페이지 열기
4. 브라우저 우측 상단 Web Clipper 아이콘 클릭
5. 템플릿 선택 → Clip → Vault 내 지정 폴더에 `.md` 저장

현재 ideaLLM Vault는 `Clippings/` 폴더에 클리핑이 들어오도록 설정돼 있다.

## ideaLLM 위키와의 연결 구조

이 프로젝트는 이미 **Schema-Wiki-Raw 3층 구조**(→ [[CLAUDE 위키 운영규칙]] = `CLAUDE.md`)로 운영 중이다. Web Clipper는 이 구조의 **자료 공급 채널** 역할을 한다.

```
웹 (블로그·뉴스·논문·유튜브)
    ↓ Web Clipper
Clippings/ (또는 raw/)  ← 사람이 채움
    ↓ "정리해줘" 요청
wiki/ (AI가 카테고리·교차참조 추가)
    ↓ 질문
답변 (위키 인용 + 자료 간 의외의 연결)
```

CLAUDE.md 운영규칙에 따라 AI는 `raw/`(또는 `Clippings/`)의 새 자료를 읽고, 기존 위키 페이지를 업데이트하거나 신규 페이지를 만들고, `index.md`·`log.md`를 갱신한다.

## 마케팅/광고 대행사 운영 관점의 활용 시나리오

| 수집 대상 | Web Clipper 저장 → 위키화 → 활용 |
|---|---|
| 업계 동향 기사 | 매주 5~10건 클리핑 → 월말 트렌드 요약 페이지 갱신 |
| 경쟁 대행사 케이스 | 클리핑 → [[enhans-회사-심층분석]] 류 페이지에 비교 항목 추가 |
| 플랫폼 정책 변화 (메타·구글·네이버) | 클리핑 → 광고 운영 가이드 페이지 누적 |
| 클라이언트 산업 리서치 | 클리핑 → 프로젝트별 페이지 백업 자료 |
| AI 도구 리뷰·튜토리얼 | 클리핑 → [[킷에이아이-프로젝트-저장소-맵]] 도구 분류 갱신 |
| 강연·세미나 후기 | 클리핑 → 인사이트 페이지로 통합 |

## AI 시대에 의미 있는 이유

LLM 응답 품질은 결국 **컨텍스트의 질**이 결정한다. Web Clipper로 매일 수집되는 자료가 위키로 정리되면, AI가 답변할 때 끌어다 쓸 수 있는 컨텍스트가 시간에 따라 복리로 증식한다. 이는 사실상 **에이전트의 장기 기억(Long-Term Memory)** 이다.

특히 마케팅 대행사처럼 **빠르게 변하는 외부 환경 + 누적되어야 하는 내부 노하우**가 공존하는 업종에서, Web Clipper + LLM 위키 조합은 다른 PKM 도구보다 ROI가 높다.

## 운영 팁

- **클리핑 위치 분리**: `Clippings/`는 자동 수집, `raw/`는 수동으로 넣는 사람 중심 자료 (회의록·PDF·메모)로 구분하면 워크플로우가 명확해짐
- **템플릿 활용**: `title`, `source`, `created`, `tags: [clippings]`를 frontmatter에 강제하면 AI가 정리 시 출처 추적이 쉬워짐
- **주 1회 일괄 정리**: 매주 금요일 "Clippings 정리해줘" 한 줄로 한 주치 자료를 카테고리별로 위키화
- **태그 일관성**: `clippings` 태그를 유지해 두면 나중에 출처별 필터링 가능
