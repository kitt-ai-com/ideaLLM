## 🔥 STEP 1 — 전환 장치 3종 추가 (마케팅 핵심, 발행보다 먼저)

> 이게 없으면 아무리 발행해도 리드가 0. **사이트 코드 수정 필요**(newsletter 저장소).

### ① 구독 폼 (이메일 수집) — 최우선
- 위치: 헤더 상단 + 각 아티클 하단 + 푸터 (3곳)
- 기능: 이메일 입력 → Supabase `subscribers` 테이블 INSERT → Resend double opt-in
- ⚠️ DB 스키마(`subscribers`)·Resend는 이미 있음 → **UI 컴포넌트만 추가하면 됨**
- 문구 예: "주 1회, 중소기업 대표를 위한 AI 활용법 — 무료 구독"

### ② 킷에이아이 브랜드 + 문의 경로
- 푸터 "© Newsletter Seoul" → **"© 2026 주식회사 킷에이아이"** + 회사 소개/문의 링크
- 헤더/About에 "이 뉴스레터는 AX 전문기업 킷에이아이가 만듭니다" 1줄
- 문의 링크 → kitt.ai.kr 또는 무료 진단 폼

### ③ 상담 전환 CTA (각 호 하단)
- 아티클 끝마다 고정 배너: **"우리 회사도 AI 도입 가능할까? → 무료 AX 진단 신청"**
- 링크: 진단 신청 폼(구글폼 or 자체) → 리드 확보

> 💡 ①②③ 모두 **newsletter 저장소의 컴포넌트/레이아웃 수정**. DB·이메일 인프라는 이미 깔려 있어 프론트 작업 위주. 1~2일 작업.

## 🔄 STEP 2 — 발행 재개 (newsletter-agent 자동화)

> 5/14 이후 멈춤 → AI 자동 초안으로 끊김 방지.

### 0. newsletter-agent 환경 확인
```bash
# newsletter-agent 저장소 (로컬에 있으면 그대로, 없으면 클론)
cd newsletter-agent && pnpm install
#  .env: GEMINI_API_KEY 또는 ANTHROPIC_API_KEY / SUPABASE service_role / AGENT_AUTHOR_ID
#  → newsletter와 같은 Supabase 프로젝트를 가리켜야 함
```

### 1단계: 호 초안 자동 생성

```bash
cd newsletter-agent
pnpm agent run        # collect→select→outline→draft→metadata 자동 실행
```
→ AI가 최신 AI 뉴스로 한국어 에세이(1,500~2,500자) 초안 생성

### 2단계: 사람 승인 (review 게이트)

- 터미널 미리보기 표시 → **승인 / 재시도 / 다른 주제 / 취소**
- ⚠️ 여기서 **킷에이아이 색깔 입히기**: AI 뉴스 큐레이션에 "그래서 중소기업은 이렇게 쓰면 된다" 한 단락 + **무료 진단 CTA** 추가
- 승인해야만 publish 호출됨

### 3단계: 발행 (publish)

- Supabase INSERT → newsletter 사이트에 즉시 반영
- covers.ts가 커버 이미지 자동 생성
- Resend로 구독자에게 발송 (newsletter 관리자에서 "발송" 버튼)

### 4단계: 운영 루틴화 (주 1회)

```
매주 월요일: pnpm agent run → 초안 생성
         ↓
화요일 오전: 승인 + CTA 추가 + 발행
```

## 🔁 첫 4주 발행 계획 (마케팅 캘린더 연동)

13주 캘린더를 newsletter 단일 채널로 축소 실행:

| 주차 | 뉴스레터 주제 | 재료(보유) |
|---|---|---|
| 1주 | AX란? 자동화와 AX의 차이 | 마케팅 플랜 포지셔닝 |
| 2주 | 물류 AX 사례 — 월 54시간 절감 | ai-agent 실적 |
| 3주 | 병원 AX 사례 — 재방문율 35%↑ | 고객 사례 |
| 4주 | AI바우처로 AI 도입하는 법 | [[접수가능-과제-2026-06]] |

> 매 호 = AI 뉴스 큐레이션(agent 자동) + 킷 케이스/인사이트 1단락(수동) + CTA

## 🔗 다른 에이전트와의 연결 (확장)

뉴스레터가 돌기 시작하면 콘텐츠를 재활용:

| 에이전트 | 활용 |
|---|---|
| **cardnews** | 뉴스레터 핵심 → 인스타/스레드 카드뉴스 자동 변환 |
| **telegram-channel** (스킬) | 뉴스레터 요약 → @kitt_AI_inc 텔레그램 발행 |
| ai-agent / ecount | 성과 수치 → 케이스 스터디 호 |

→ **하나의 콘텐츠(뉴스레터) → 다채널 배포** 구조. 먼저 뉴스레터부터 안정화.
