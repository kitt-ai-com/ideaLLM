# 킷에이아이 고객 사례 리패키징 가이드

## 인핸스의 도입 사례 공식

인핸스가 비즈니스 케이스를 구성하는 공식:

```
[타이틀] {업종} 기업이 {에이전트명}으로 {핵심 성과}한 방법

[구조]
1. Overview — 한 줄 요약 (누가 + 무엇을 + 어떤 성과)
2. Industry Challenges — 해당 업종의 4가지 문제점
3. Solution — 에이전트가 해결하는 방법 (다이어그램)
4. How It Works — 3단계 프로세스
5. Business Outcomes — ROI 수치 + 고객 증언
6. CTA — 무료 상담 신청
```

## 킷에이아이 기존 성과 → 에이전트 도입 사례로 변환

### 사례 1: 물류 출고 자동화 에이전트

**현재 성과 데이터:**
- 매일 3시간 → 15분으로 단축
- 월 54시간 절감

**리패키징 결과:**

#### 타이틀
> "물류 기업이 kitt 출고 자동화 에이전트로 월 54시간을 절감한 방법"

#### Overview
물류/유통 기업이 매일 반복되는 출고 정리 업무를 kitt의 출고 자동화 에이전트로 전환하여, 월 54시간을 절감하고 운영 비용을 35% 줄인 사례입니다.

#### Industry Challenges (물류/유통 업종)
| # | 문제 | 설명 |
|---|---|---|
| 01 | 수작업 출고 정리 | 매일 3시간씩 수동으로 출고 데이터를 정리하며 인력 낭비 |
| 02 | 데이터 입력 오류 | 수작업 과정에서 발생하는 오류로 배송 지연, 고객 불만 |
| 03 | 실시간 현황 파악 불가 | 출고 현황을 즉시 확인할 수 없어 의사결정 지연 |
| 04 | 인력 의존 구조 | 담당자 부재 시 업무 마비, 확장 시 인력 비례 증가 |

#### How It Works
```
1단계: 데이터 수집    — 주문/재고/물류 시스템에서 자동 데이터 동기화
2단계: AI 분류/정리   — 출고 건별 자동 분류, 우선순위 지정, 오류 검출
3단계: 보고/실행      — 출고 리포트 자동 생성, 담당자 알림, 이상 건 플래그
```

#### Business Outcomes
| 지표 | Before | After | 개선 |
|---|---|---|---|
| 일일 출고 정리 시간 | 3시간 | 15분 | **92% 단축** |
| 월간 절감 시간 | - | 54시간 | **연 648시간** |
| 데이터 오류율 | 수작업 오류 | 0% | **오류 제거** |
| 운영 비용 | 기준 | -35% | **35% 절감** |

#### 고객 증언 (예시 — 실제 고객 확인 필요)
> "매일 아침 3시간씩 엑셀과 씨름하던 출고 정리가 이제 15분이면 끝납니다. 그 시간에 더 중요한 일에 집중할 수 있게 되었습니다."

#### 추가 환산 수치 (영업용)
- 월 54시간 × 시급 2만원 = **월 108만원 인건비 절감**
- 연간 **약 1,300만원 절감** (1인 기준)
- 담당자 2~3명 기준: **연 2,600~3,900만원 절감**

### 사례 2: 병원 환자관리 에이전트

**현재 성과 데이터:**
- 재방문율 35% 향상

**리패키징 결과:**

#### 타이틀
> "클리닉이 kitt 환자관리 에이전트로 재방문율을 35% 높인 방법"

#### Industry Challenges (의료/클리닉 업종)
| # | 문제 | 설명 |
|---|---|---|
| 01 | 환자 이탈 | 초진 후 재방문 없이 이탈하는 환자 비율 높음 |
| 02 | 수동 리마인드 | 직원이 일일이 전화/문자로 예약 확인, 시간 소모 |
| 03 | 개인화 부재 | 환자별 맞춤 케어 메시지 불가, 일괄적 안내 |
| 04 | 데이터 활용 미흡 | 환자 데이터가 있어도 분석/활용하지 못함 |

#### How It Works
```
1단계: 환자 데이터 통합  — EMR/예약 시스템 연동, 환자별 히스토리 구축
2단계: AI 분석/추천     — 재방문 가능성 예측, 맞춤 케어 메시지 생성
3단계: 자동 커뮤니케이션 — 적시에 맞춤 리마인드/안내 자동 발송
```

#### Business Outcomes
| 지표 | Before | After | 개선 |
|---|---|---|---|
| 환자 재방문율 | 기준 | +35% | **35% 향상** |
| 직원 리마인드 업무 | 수동 | 자동 | **업무 해방** |
| 환자 만족도 | 기준 | 향상 | **맞춤 케어** |

#### 추가 환산 수치 (영업용)
- 평균 진료비 5만원 × 재방문 환자 증가분
- 월 환자 200명 기준, 35% 재방문 증가 = **월 70건 추가 진료**
- **월 350만원, 연 4,200만원 추가 매출**

### 사례 3: 학원 운영 자동화 에이전트

**현재 성과 데이터:**
- 업무 효율 3배 향상

**리패키징 결과:**

#### 타이틀
> "학원이 kitt 운영 에이전트로 업무 효율을 3배 높인 방법"

#### Industry Challenges (교육/학원 업종)
| # | 문제 | 설명 |
|---|---|---|
| 01 | 행정 업무 과부하 | 원장이 교육보다 행정(수납, 스케줄, 학부모 소통)에 시간 소모 |
| 02 | 수강생 관리 한계 | 출결, 성적, 상담 기록 수작업 관리 |
| 03 | 학부모 소통 부담 | 개별 알림/상담/피드백 전달에 많은 시간 |
| 04 | 데이터 기반 운영 불가 | 감에 의존한 의사결정 (반 편성, 강사 배치 등) |

#### How It Works
```
1단계: 운영 데이터 통합  — 수강/출결/성적/수납 데이터 자동 수집
2단계: AI 자동화         — 스케줄 관리, 학부모 알림, 리포트 자동 생성
3단계: 인사이트 제공     — 수강생 이탈 예측, 반 편성 추천, 매출 분석
```

#### Business Outcomes
| 지표 | Before | After | 개선 |
|---|---|---|---|
| 업무 효율 | 기준 | 3배 | **300% 향상** |
| 행정 업무 시간 | 기준 | 대폭 감소 | **교육 집중 가능** |
| 학부모 소통 | 수동 | 자동 | **만족도 향상** |

## 리패키징 산출물 체크리스트

### 웹사이트용 (각 에이전트별)
- [ ] 비즈니스 케이스 랜딩페이지 (인핸스 공식 따름)
- [ ] 대표 이미지/다이어그램
- [ ] ROI 수치 인포그래픽

### 영업용
- [ ] 1페이지 원페이저 PDF (에이전트별)
- [ ] 상세 케이스 스터디 PDF (3~5페이지)
- [ ] 제안서 템플릿 (에이전트별)

### 투자 IR용
- [ ] 3개 사례 통합 슬라이드 (5장)
- [ ] TAM/SAM/SOM 시장 규모 추정
- [ ] MRR/ARR 시뮬레이션

### 콘텐츠 마케팅용
- [ ] 블로그 포스트 (에이전트별 도입기)
- [ ] SNS 카드뉴스 (성과 수치 중심)
- [ ] 웨비나 발표 자료

## 실행 일정 (1주일)

| 일차 | 작업 |
|---|---|
| Day 1~2 | 기존 고객에게 성과 수치 재확인, 증언 요청 |
| Day 3~4 | 비즈니스 케이스 3건 작성 (인핸스 공식) |
| Day 5 | 원페이저 PDF 3종 제작 |
| Day 6 | 웹사이트 에이전트 페이지 초안 작성 |
| Day 7 | 검토/수정/게시 |
