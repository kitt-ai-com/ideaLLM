## 1. 과제 해결방안 (개발계획)

### 1-1. 해당 과제를 풀기 위한 전반적인 개발계획

**◦ 개발 배경**
- 복잡한 정책자금 신청서 작성은 인적·시간적 여유가 부족한 중소기업에 진입장벽으로 작용 (작성 1건당 평균 3시간 이상)
- 작성 경험 부족 → 부실·과장 기재 또는 불법 브로커 의존 → 불필요한 비용·리스크 발생
- **AI가 신청서 초안을 생성**하여 기업 스스로 정확하게 작성하는 환경을 조성, 제3자 부당개입을 구조적으로 차단

**◦ 개발 목적**
- 중소기업 대표가 **키워드 5~10개만 입력하면 30초 안에** 중진공 공식 서식에 맞는 신청서 초안을 자동 생성하는 반응형 웹서비스(kitt PolicyDoc Agent) 구축

**◦ 핵심 기술 및 구조 — 3원칙**
- **Fact-Grounded 생성**: 중진공 공식 공고문·서식·예시만 RAG 소스로 사용 → 외부·허위 정보 생성 불가
- **서식 완전 보존**: HWP/PDF 원본 레이아웃·폰트·여백 100% 유지 → 별도 편집 없이 제출 가능
- **Zero Data Retention**: 다운로드 완료 즉시 Secure Delete → 개인정보 잔류 없음

**◦ 서비스 시나리오 (6단계)**
- ① SSO 로그인(중진공 고객포털 연동, 기업 기본정보 자동 매핑) → ② 자금유형 선택 + 키워드 입력 → ③ AI 초안 생성(30초 이내, 항목별 스트리밍, 문장별 출처 태깅) → ④ 웹에디터 항목별 수정·재생성(최대 5회/항목) → ⑤ 미리보기 + HWP/PDF 다운로드 → ⑥ 다운로드 즉시 임시데이터 Secure Delete

**◦ 핵심 차별점 — 이미 운영 중인 파이프라인의 도메인 이전**
- 킷에이아이는 동일 구조의 AI 문서생성 파이프라인 **newsletter-agent를 현재 운영 중**(GitHub, 최신 2026-05-18)
- collect→select→outline→draft→metadata→review→publish 7단계가 본 과제(공공데이터 파싱→서식분류→항목구조화→초안생성→정보추출→웹검토→HWP출력)와 1:1 대응
- **신규 개발이 아니라 검증된 파이프라인의 도메인 변환** → 개발 리스크 최소화

### 1-2. 개발 추진일정

**◦ 10주 개발 로드맵 (본선 진출 시, 6.29~)**

| 주차 | 핵심 작업 | 산출물 |
|---|---|---|
| 1주 | 중진공 제공 데이터 수령·파싱·정제, KT Cloud(CSAP) 환경 구축 | 데이터 정제 완료, 인프라 세팅 |
| 2주 | 벡터화·인덱싱·서식 분류 엔진 구축 | 벡터DB 구축 |
| 3주 | RAG + 프롬프트 엔지니어링 + Zod 스키마 설계 | 초안 생성 엔진 v1 |
| 4주 | 할루시네이션 3중 방어 구조 적용·검증 | 출처 태깅 + Fallback |
| 5주 | SSO 연동 + 웹에디터 + 재생성 구현 | 베타 UI |

| 7주 | 50인 동시 부하 테스트 + 성능 튜닝 | 정량목표 6개 달성 확인 |

**◦ 일정 단축 근거**: newsletter-agent의 collect·draft·validate·review·publish 모듈을 직접 재사용 → 1~2주차 파이프라인 구축 기간이 타 팀 대비 절반 이하. 확보된 2~3주 여유를 UAT·발표 준비에 추가 투입.

## 2. 혁신성

### 2-1. 사용된 AI 알고리즘 및 AI 파운데이션 모델 추진체계

**◦ AI 파운데이션 모델**
- **Claude API (Anthropic)** — 한국어 장문 생성 최상위 성능, 킷에이아이 **실운영 경험 보유**
- LLM 추상화 인터페이스(`LlmClient`)로 Claude↔Gemini 공급사 교체를 1개 파일 수정으로 대응 (newsletter-agent에서 검증)

**◦ 핵심 알고리즘 — 할루시네이션 3중 방어 (차별성)**

| 방어층 | 구현 | 효과 |
|---|---|---|
| 1층: RAG 소스 제한 | 중진공 공식 문서만 인덱싱, 외부 URL 차단 | 공식 범위 외 생성 물리적 불가 |
| 2층: 출처 태깅 | 생성 문장마다 참조 서식 번호·조항 병기 | 심사자가 출처 즉시 검증 |
| 3층: Fallback | 확신도 임계값 미달 항목 → "직접 입력" 안내 | 불확실한 내용 강행 없음 |

**◦ 추진체계**
- RAG 파이프라인: 중진공 공고문·서식·예시 → 벡터 검색 + 컨텍스트 주입
- LLM 초안 생성: Claude + Few-Shot + CoT + 항목별 스트리밍(첫 응답 5초 이내)
- **Zod 스키마 검증**: LLM 출력 타입 강제, 실패 시 위반 메시지 포함해 **3회 자동 재시도**, 그래도 실패 시 파이프라인 중단(잘못된 출력 차단)
- 사람 승인 게이트: 자동 게시 경로 없음, 반드시 사용자 검토 후 출력

**◦ 혁신성·차별성 (타 참가팀 대비)**
- 타 팀: "이런 시스템을 만들겠다"(계획) / 킷에이아이: **동일 구조 파이프라인이 GitHub에서 운영 중**(실증)
- 병원 환자 데이터 Zero Data Retention 실운영 실적 → 공공 개인정보 처리 신뢰성 입증

### 2-2. 최종 완성될 결과물의 성능 목표 (정량 지표)

| 목표 항목 | 수치 | 측정 방법 |

## 3. 가치창출 및 시장성

### 3-1. 현안 해결 노력

**◦ 과제 출제 배경에 대한 해결 (중진공 현안 직접 대응)**
- 출제 배경 ①"신청서 작성 진입장벽" → 키워드 입력만으로 30초 초안 생성, 작성시간 70% 단축
- 출제 배경 ②"부실·과장 기재, 불법 브로커" → 공식 문서 RAG + 출처 태깅으로 사실 기반 작성, 기업 직접 작성 환경 조성 → **제3자 부당개입 차단**

**◦ 기업 관점 기대효과**

| 항목 | 현재 | 도입 후 | 개선 |
|---|---|---|---|
| 작성 시간 | 평균 3시간/건 | 30~50분 | 70~83% 단축 |
| 대행 비용 | 건당 수십~수백만원 | 0원 | 비용 완전 제거 |
| 접근성 | 영세기업 낮음 | 키워드만 있으면 신청 | 장벽 해소 |
| 리스크 | 대행 사실 왜곡 가능 | 기업 직접 작성 | 리스크 제거 |

**◦ 중진공 관점 기대효과**
- 초안 완성도 향상 → 보완 요청 반복 감소 / 동일 문의 자동가이드 → 응대 부담 경감 / 직접작성 환경 → 부당개입 검증 부담 경감 / 정부 AI 전환 기조 선도

**◦ 실증 근거 (계획이 아닌 실측)**

- 병원 환자관리: 재방문율 **35%** 향상 / 학원: 운영효율 **3배**

### 3-2. AI 모델의 경제성 및 확장 방향

**◦ 운영비용 대비 효과 (경제성)**
- 연간 정책자금 신청 약 20,000건 × 건당 2시간 이상 절감 = 연 **40,000시간+** 절감 → 사회적 가치 약 **8억원/년**(시간당 2만원 기준)
- LLM API 비용은 프롬프트 캐싱·토큰 최적화(newsletter-agent 실적)로 통제, 건당 운영비 대비 절감효과 압도적

**◦ 서비스 확산·확장 (3단계)**
- **Phase 1 (우승 즉시, 2026 하반기)**: 중진공 솔루션 구매계약 → 고객포털 정식 서비스 → 첫 공공 레퍼런스 확보
- **Phase 2 (2027 상반기)**: 소진공·창진원 등 타 공공기관 확장 → 동일 파이프라인 도메인 이전(개발비 최소) → 기관별 서식팩 SaaS 구독
- **Phase 3 (2027 하반기)**: 민간 금융기관 대출신청서·기업 IR 자료 자동화 → B2B SaaS 월정액(MRR 전환)
- 목표 매출: 중진공 유지·운영 연 5,000만원+ / 공공 2곳 추가 연 3,000만원 / 민간 SaaS 10사 월 500만원 MRR
