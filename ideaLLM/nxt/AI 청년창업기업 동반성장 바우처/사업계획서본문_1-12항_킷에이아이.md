## 7. 과제추진계획

```
[추진 방법]

■ 제로 코딩 AI 도입 전략
  수요기업 기존 업무 프로세스(엑셀·카카오톡 주문)를 변경하지 않고
  AI가 기존 데이터를 그대로 인식·처리하는 방식 적용.
  - 엑셀 양식 변경 불필요: 어떤 형태의 주문 엑셀도 LLM이 자동 해석
  - 카카오톡 주문 유지: 이미지 캡처 → OCR+LLM 자동 추출로 기존 채널 그대로 활용
  - 시스템 교체 없음: API 연동 방식으로 기존 업무 도구에 AI 레이어 추가

■ 이중 안전망 구조 (무중단 운영 보장)
  LLM 1차 처리 → confidence 점수 기반 운영자 검토 →
  규칙 기반 폴백의 이중 구조로 AI 오작동 시에도 업무 연속성 보장.
  - confidence 임계값(기본 0.85) 이상: 자동 처리
  - 임계값 미만: 운영자 검토 플래그 자동 표시
  - LLM 장애 시: 규칙 기반 폴백 엔진 자동 전환 (0초 다운타임)

■ 멀티테넌트 SaaS 즉시 확장 구조
  수요기업 1개사 도입 후 추가 기업 즉시 확장 가능한 멀티테넌트 구조.
  테넌트별 데이터 완전 격리로 보안 및 개인정보 보호 동시 달성.

[단계별 추진계획]

1단계 (7월 1~2주): 수요기업 환경 분석 및 요구사항 정의
  - 현행 프로세스 인터뷰: 주문 접수→처리→배송→정산 전 흐름 파악
  - 사용 중인 엑셀 양식 수집 (채널별 최소 3개월 이상 실사용 데이터)
  - 공급사 정산 조건·수수료 체계 수집
  - 도입 전 기준 지표 측정: 처리 시간, 오류율, 정산 소요 시간

2단계 (7월 2~3주): 테넌트 설정 및 초기 데이터 구축
  - 수요기업 전용 테넌트 생성 및 관리자 계정 발급
  - 상품 DB 초기 등록 (대표 상품 50종 이상)
  - 공급사 계정 생성 및 정산 조건 설정
  - 셀러(판매 채널) 계정 생성 및 배송정책 등록

3단계 (7월 3주~8월): AI 엑셀 매핑 커스터마이징 및 정확도 향상
  - 수요기업 실사용 엑셀 양식 기반 LLM 매핑 프롬프트 커스터마이징
  - confidence 임계값 조정 (수요기업 허용 오류율 기준 최적화)
  - 이미지 주문 추출 테스트 (카카오톡 실제 주문 이미지 100건 이상)
  - 배송비 정책 등록 및 자동 산정 로직 검증

4단계 (8~9월): 파일럿 운영 및 성과 검증
  - 전체 기능 파일럿 운영 (실제 주문 처리에 셀킷 병행 적용)
  - 도입 전후 처리 시간·오류율 비교 측정 (주 1회 보고)
  - 운영자 교육 실시: 관리자(2시간) + 셀러 담당자(1시간)
  - 정산 자동화 1차 검증 (파일럿 기간 정산 결과 vs 수작업 결과 대조)

5단계 (10~12월): 전면 전환 및 성과 관리
  - 수작업 → 셀킷 전면 전환 (이관 완료 선언)
  - AI 이상탐지 엔진 가동 (주문 급증/급감, 반품 이상, 재고 소진 실시간 모니터링)
  - 월 1회 정기 성과 보고 (처리 시간, 정확도, 클레임률 추이)
  - 최종 성과 보고서 작성 및 제출

[협력 체계]
  킷에이아이 (공급기업) ↔ 수요기업 담당자 간 슬랙/카카오톡 전용 채널 운영
  주 1회 정기 미팅 (온라인) + 월 1회 현장 방문 (서울 수도권 기준)
  24시간 장애 대응 체계 운영 (3시간 내 1차 응답, 24시간 내 해결)
```

## 8. 과제 내용

```
[사업 범위]
본 과제는 킷에이아이의 AI 솔루션 "셀킷(SellKitt)"을 수요기업에 공급·구축하고
6개월간 운영·검증하는 사업이다.
셀킷은 LLM 기반 엑셀 자동 매핑부터 이미지 주문 추출, 배송비 산정,
공급사 정산, AI 챗봇, 이상탐지까지 B2B 도매·유통 업무 전 과정을 AI로 자동화한다.

[주요 기능별 상세 내용]

① AI 엑셀 컬럼 자동 매핑
   - Gemini 2.5 Flash LLM이 어떤 양식의 주문 엑셀도 컬럼 헤더+샘플 데이터 동시 분석
   - 17종 이상 판매채널(스마트스토어, 쿠팡, 11번가, 위메프, G마켓 등) 양식 대응
   - confidence 점수(0~1)로 불확실 항목 자동 플래그 → 운영자 최종 확인
   - LLM 실패 시 규칙 기반 폴백 엔진 자동 전환 (무중단)
   - 내부 테스트 정확도: 87% (→ 과제 기간 내 90% 이상 달성 목표)

② 이미지 주문 자동 추출
   - 카카오톡 캡처 이미지 업로드 → OCR + LLM으로 수령자·상품·수량 자동 추출
   - 건당 수기입력 5분 → 30초로 단축 (90% 절감)
   - 지원 이미지 형식: JPG, PNG, 카카오톡 캡처 (저해상도 이미지도 처리 가능)
   - 추출 실패 시 원본 이미지와 함께 수동 입력 화면 자동 제공

③ 지능형 배송비 자동 산정
   - 4가지 배송비 규칙 유형 지원:
     (1) 수량별 구간 요금 (예: 1~5개 3,000원 / 6~10개 5,000원)
     (2) 무게별 구간 요금
     (3) 주문금액 기준 무료배송 조건
     (4) N개마다 M원 반복 규칙 + 합배송 자동 계산
   - 수요기업 현행 배송정책 그대로 시스템에 반영 (별도 재설계 불필요)

④ 공급사 자동 정산 워크플로우
   - 매출·수수료·클레임 차감·VAT 자동 집계
   - 정산 완료 시 카카오 알림톡 자동 발송 (공급사 즉시 통보)
   - 알림톡 5종: 주문접수·송장등록·반품접수·정산확정·정산지급 완료
   - 정산 내역서 자동 생성 (PDF 다운로드 지원)

⑤ AI 운영 챗봇 (22개 Function Calling Tool)
   - 자연어로 주문조회·상태변경·상품검색·매출통계·클레임 처리
   - Function Tool 22개 예시:
     주문조회(날짜/채널/상태별), 주문상태변경, 상품검색/등록/수정,
     재고조회, 배송조회, 반품접수/처리, 정산내역조회, 매출통계, 클레임관리 등
   - 비IT 담당자도 즉시 사용 가능 (교육 없이 자연어 입력)

⑥ AI 이상탐지 엔진
   - 주문 급증/급감: 전주 동일 요일 대비 ±70% 이상 변동 시 자동 경보
   - 반품률 이상: 채널별 반품률이 기준치(5%) 초과 시 경보
   - 재고 소진: 상품 재고가 안전재고 이하로 감소 시 경보
   - 발주 미수신: 7일 이상 신규 주문 미접수 시 경보
   - 경보 수신: 관리자 이메일·카카오톡 즉시 발송

[기술 스택]
   - 프론트엔드: Next.js 16 + TypeScript (서버사이드 렌더링, SEO 최적화)
   - 백엔드: Next.js API Routes + Prisma 7 ORM
   - 데이터베이스: PostgreSQL (Railway, 멀티테넌트 격리)
   - AI 엔진: Google Gemini 2.5 Flash (LLM + Vision)
   - 인프라: Vercel (글로벌 CDN, 자동 스케일링)
   - 결제: Toss Payments
   - 알림: 카카오 알림톡 API
```

## 9. 수요기업 수행역량

```
[도입 환경 및 필요성]
수요기업은 스마트스토어·쿠팡·카카오톡·자사몰 등 4개 이상 채널을 동시 운영하며
채널별 주문 양식 차이로 인해 일 평균 2~4시간의 수작업 처리가 발생하고 있다.

  현황 분석:
  - 채널별 엑셀 양식 상이 → 매번 수동으로 컬럼 매핑 및 데이터 정리
  - 카카오톡 이미지 주문 건당 5분 수기 입력 → 하루 10건 이상 시 50분 소요
  - 오입력·오배송 클레임 월 3~4% 발생 → 처리 비용 및 거래처 신뢰도 저하
  - IT 전담 인력 없음 → 외부 ERP 도입 시 유지보수 불가

  셀킷 도입 필요성:
  - 기존 채널·양식 변경 없이 AI가 자동 처리 → 즉시 도입 가능
  - 비IT 담당자도 교육 없이 운영 가능 → 내부 인력으로 완전 자립
  - 클라우드 SaaS 방식 → 별도 서버·유지보수 불필요

[데이터 보유 현황]
AI 솔루션 즉시 적용을 위한 학습·검증 데이터를 충분히 보유하고 있다.

  - 판매채널별 주문 엑셀: 최근 12개월 이상 채널별 주문 데이터 보유
    (AI 매핑 정확도 학습·검증 기준 데이터로 활용)
  - 공급사 정산 이력: 최근 6개월 이상 정산 엑셀 보유
    (정산 자동화 검증 기준 데이터로 활용)
  - 카카오톡 주문 이미지: 최근 3개월 이미지 주문 캡처 보유
    (OCR+LLM 추출 정확도 검증 데이터로 활용)

[AI 도입 준비도]
  - 도입 의지: 수요기업 대표가 AI 전환(AX)을 2026년 핵심 경영 목표로 선정
  - 업무 표준화 선행: 채널별 주문 처리 표준 절차 수립 완료
  - 데이터 접근성: 모든 채널 주문 데이터를 CSV/엑셀로 즉시 추출 가능
  - IT 환경: 윈도우 PC + 크롬 브라우저 기반 → 셀킷 웹 플랫폼 즉시 접속 가능
  - 내부 수용성: 담당자 1인이 전담 운영 예정, AI 솔루션 학습 의지 높음

[참여인력 구성]
  - 총괄책임자: 수요기업 대표 또는 운영 총괄 담당자
    (의사결정, 예산 집행, 과제 진행 총괄)
```
