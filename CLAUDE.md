# LLM 나무위키 — Schema

> 이 파일은 위키의 운영 규칙(Schema)입니다. AI는 이 규칙을 따라 위키를 관리합니다.

## 프로필

- **역할**: 마케팅/광고 대행사 운영자
- **위키 목적**: 업무와 학습에서 쌓이는 지식을 복리로 축적하는 나만의 지식 베이스
- **축적 자료**: 업계 동향, 경쟁사 분석, 프로젝트/회의 기록, 독서 인사이트, 강연/교육 노하우, 개인 프로젝트 기록

## 3층 구조

| 층 | 위치 | 역할 | 관리 주체 |
|---|---|---|---|
| Schema | `CLAUDE.md` | 위키 운영 규칙 정의 | 사람이 설정 |
| Wiki | `wiki/` 폴더 | AI가 원본을 읽고 정리한 지식 페이지 | AI가 작성·관리, 사람은 읽기만 |
| Raw | `raw/` 폴더 | 기사, PDF, 회의록, 메모 등 원본 자료 | 사람이 넣고, AI는 읽기만 |

## 폴더 규칙

- `raw/`와 `wiki/` 안에는 **하위 폴더 없이 플랫하게** 관리
- 분류는 `wiki/index.md`에서 카테고리별로 정리
- 위키 페이지는 마크다운(`.md`)으로 작성
- 페이지 간 `[[위키링크]]`로 교차참조

## 3가지 운영 방법

### 1. 넣기 (Ingest)

사용자가 `raw/` 폴더에 자료를 넣고 "정리해줘"라고 요청하면:

1. `raw/` 폴더의 새로운(미처리) 자료를 읽는다
2. 각 자료에서 핵심 개념, 인물, 도구, 프레임워크 등을 추출한다
3. 기존 위키 페이지가 있으면 업데이트, 없으면 새로 생성한다
4. **모든 위키 페이지 맨 앞에 callout 블록**으로 "내 비즈니스 관점에서의 핵심 takeaway"를 넣는다:
   ```
   > [!takeaway] 핵심 테이크어웨이
   > 마케팅/광고 대행사 운영 관점에서 이 내용의 핵심은...
   ```
5. 관련 기존 페이지와 `[[위키링크]]`로 교차참조한다
6. **서로 다른 자료 간 의외의 연결점을 적극적으로 연결**한다
7. `wiki/index.md`를 업데이트한다
8. `wiki/log.md`에 작업 이력을 기록한다

**위키 페이지 템플릿:**
```markdown
> [!takeaway] 핵심 테이크어웨이
> (내 비즈니스/방향성 관점에서의 핵심 내용)

# 페이지 제목

## 개요
(핵심 내용 요약)

## 상세 내용
(구체적 내용)

## 관련 페이지
- [[관련 페이지1]]
- [[관련 페이지2]]

## 출처
- 원본: `raw/파일명.md`
- 처리일: YYYY-MM-DD
```

### 2. 질문하기 (Query)

사용자가 질문하면:

1. `wiki/` 폴더의 정리된 페이지에서 답변한다 (매번 원본을 뒤지지 않는다)
2. 답변에 근거가 되는 위키 페이지를 `[[위키링크]]`로 인용한다
3. 좋은 답변이 나오면 해당 내용을 위키 페이지로 만들어 지식을 축적한다
4. 교차 질문(서로 다른 자료 간 연결)에 특히 주의를 기울인다

### 3. 건강검진 (Lint)

사용자가 "위키 점검해줘"라고 요청하면:

1. 페이지 간 모순이 없는지 확인
2. 오래된 정보 플래그 표시
3. 빠진 내용이나 고아 페이지(참조되지 않는 페이지) 확인
4. `[[위키링크]]`가 깨진 곳이 없는지 확인
5. index.md가 최신 상태인지 확인
6. 결과를 `wiki/log.md`에 기록

## 카테고리 (index.md 분류용)

- **업계 동향**: 마케팅/광고 업계 트렌드, 플랫폼 변화, 시장 동향
- **경쟁사 분석**: 경쟁 대행사, 경쟁 브랜드 분석
- **클라이언트/프로젝트**: 프로젝트 기록, 회의록, 의사결정 기록
- **전략/프레임워크**: 마케팅 전략, 비즈니스 프레임워크, 멘탈모델
- **도구/기술**: 마케팅 도구, AI 도구, 업무 자동화
- **인사이트**: 독서 인사이트, 강연/교육에서 배운 내용
- **인물**: 업계 주요 인물, 전문가

## 파일 네이밍 규칙

- 위키 페이지: `wiki/주제-이름.md` (한글, 소문자, 하이픈 구분)
- 원본 자료: `raw/YYYYMMDD-출처-제목.md` 또는 사용자가 넣는 그대로
- index: `wiki/index.md`
- 작업 로그: `wiki/log.md`

## 중요 원칙

1. **Raw는 읽기 전용**: AI는 `raw/` 폴더의 파일을 절대 수정하지 않는다
2. **Wiki는 AI 관리**: 사람은 `wiki/`를 읽기만 하고, 수정은 AI가 한다
3. **교차참조 적극 활용**: 새 페이지를 만들 때 반드시 기존 페이지와 연결점을 찾는다
4. **비즈니스 관점 우선**: 모든 정리는 마케팅/광고 대행사 운영 관점에서의 실용적 시사점을 먼저 제시한다
5. **복리 효과**: 자료가 쌓일수록 교차참조가 풍부해지고, 질문 답변 품질이 올라간다
